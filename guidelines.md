# Flixses Suites Website Design Guidelines

## Project Overview

- **Brand name:** Flixses Suites
- **Industry:** Luxury hospitality
- **Positioning:** Elegant, modern, minimal, premium suites experience
- **Design direction:** Flat luxury minimalism with refined typography, disciplined spacing, and restrained gold accents

---

## Core Brand Direction

Flixses Suites should feel:

- Luxurious
- Minimal
- Editorial
- Calm
- Premium
- Modern
- Spacious
- Refined

The interface should communicate **quiet confidence**, not loud luxury.

---

## Logo Usage

### Primary Logo Style
Use the flat monogram/logo with:

- White background
- Black `F`
- Gold curved `S` accent

### Approved Backgrounds

- `#FFFFFF`
- `#F8F6F2`
- `#111111`

### Clear Space
Keep space around the logo equal to at least the width of the gold accent stroke.

### Minimum Sizes

- **Desktop header:** 140px width
- **Mobile header:** 110px width
- **Favicon exports:** 32x32, 48x48, 180x180, 512x512

### Do Not

- Do not add gradients, bevels, shadows, or 3D effects
- Do not stretch or distort the logo
- Do not place the logo on noisy backgrounds without overlay control
- Do not recolor the gold accent to arbitrary colors
- Do not overdecorate the mark with frames or ornaments

### Primary Colors

- **Warm White:** `#F8F6F2`

- Use **black** for main text, headings, navigation, icons, and dark sections
- Use **gold** only as an accent for highlights, hover states, dividers, premium details, and subtle emphasis

- Avoid introducing extra accent colors unless there is a strong product reason

## Typography

### Heading Fonts
Use an elegant high-contrast serif.

Recommended options:

- Cormorant Garamond
- Playfair Display
- Bodoni Moda

Fallback: `serif`

### Body Fonts
Use a clean sans-serif.

Recommended options:

- Inter
- Manrope
- Plus Jakarta Sans

Fallback: `sans-serif`

### Type Scale

- **H1:** 56–72px desktop, 38–46px mobile
- **H2:** 40–52px desktop, 30–36px mobile
- **H3:** 28–36px desktop, 24–28px mobile
- **Body large:** 20px
- **Body:** 16–18px
- **Small:** 14px
- **Caption:** 12px

### Typography Rules

- Use serif typography for hero headings and premium messaging
- Use sans-serif for body text, labels, forms, navigation, and UI controls
- Use generous tracking for small uppercase labels
- Do not rely on ultra-bold weights for luxury styling
- Let spacing and hierarchy do the work

---

## Layout System

### Grid

- 12-column grid on desktop
- 6-column grid on tablet
- 4-column grid on mobile

### Container

- **Max content width:** 1280px

### Spacing

- **Section spacing:** 96–140px desktop
- **Section spacing:** 64–88px mobile

### Layout Behavior

- Use generous white space
- Prefer balanced asymmetry over rigid symmetry
- Keep visual rhythm consistent between sections
- Avoid cramped layouts and crowded cards

### Radius and Depth

- Border radius should stay between `0px` and `12px`
- Prefer subtle borders and contrast over heavy shadows
- Any shadows used should be extremely soft

---

## UI Style

### General Principles

- Keep everything flat and minimal
- Luxury should come from typography, photography, spacing, and restraint
- Use gold sparingly
- Avoid busy interfaces
- Avoid dashboard-style clutter unless functionally required

### Cards

- **Background:** `#FFFFFF`
- **Border:** `1px solid #EAE3D8`
- **Radius:** `10px`
- **Padding:** `24px–32px`

### Dividers

- Use thin gold or beige lines
- Thickness should usually be `1px`

### Icons

- Use simple, thin, minimal icons
- Black is primary
- Gold is accent only

### Buttons

#### Primary Button
- Background: `#111111`
- Text: `#FFFFFF`
- Border: `1px solid #111111`
- Hover: gold background with dark text

#### Secondary Button
- Background: transparent
- Text: `#111111`
- Border: `1px solid #111111`
- Hover: black background with white text

#### Accent Button
- Background: `#B08A3E`
- Text: `#111111`
- Border: `1px solid #B08A3E`
- Hover: `#C5A76A`

### Form Inputs

- Background: `#FFFFFF`
- Border: `1px solid #D9D4CC`
- Focus border: `#B08A3E`
- Text: `#111111`
- Placeholder: `#8A857D`
- Radius: `8px`

## Imagery Direction

### Photography Style

Use luxury editorial hospitality photography with:

- Natural light
- Warm neutral tones
- Premium interiors
- Architectural framing
- Clean bedding and materials
- Marble, brass, stone, glass, wood, and textured fabrics
- Calm, uncluttered compositions

### Avoid

- Oversaturated travel imagery
- Cheap-looking stock photos
- Crowded spaces
- Aggressive filters
- Generic business visuals

### Image Treatment

- Soft overlay only when needed for text contrast
- Prefer square corners or very subtle rounding
- Recommended aspect ratios:
  - `16:9`
  - `4:5`
  - `3:2`

## Motion and Interactivity

### Animation Philosophy

Motion should feel:

- Smooth
- Light
- Premium
- Intentional

Do not use flashy or excessive animation.

### Approved Motion Types

- Fade in
- Slide up
- Gentle reveal
- Soft hover transitions
- Subtle parallax only if performance stays excellent

### Timing

- Use durations between `200ms` and `500ms`
- Keep easing smooth and natural
- Motion should support clarity, not distract from it

## Frontend Stack Rules

### Required Technologies

When building this website, use:

- **Astro**
- **Tailwind CSS v4**
- **Alpine.js** for lightweight UI state when needed
- **GSAP** only for refined motion or section transitions when needed

### Tailwind CSS v4 Rules

- Use Tailwind CSS v4 conventions and best practices
- Prefer utility-first styling
- Extract repeated patterns into components or reusable class strategies
- Use design tokens consistently for spacing, colors, typography, and section rhythm
- Avoid arbitrary values unless truly necessary
- Keep class lists readable and maintainable

### Alpine.js Rules

Use Alpine.js only where lightweight client-side interaction is needed, such as:

- Mobile navigation
- FAQ accordion
- Filter toggles
- Booking UI helpers
- Simple modal states

Do not use Alpine where plain HTML or Astro islands are enough.

### GSAP Rules

Use GSAP only when subtle premium motion adds value, such as:

- Hero entrance animation
- Scroll-triggered reveal sequences
- Elegant image transitions
- Section choreography for storytelling pages

Do not use GSAP for decorative animation overload.
Always prioritize performance, accessibility, and reduced-motion support.

---

## Astro Best Practices

### Architecture

- Follow Astro-first architecture
- Prefer static output where possible
- Hydrate only where interactivity is actually needed
- Keep JavaScript payload minimal
- Default to server-rendered or static content first

### Component Rules

- Build reusable components for:
  - Header
  - Footer
  - Hero blocks
  - CTA sections
  - Suite cards
  - Testimonials
  - Blog cards
  - SEO helpers

### Content Rules

- Use Astro content collections for blog posts, guides, or structured content
- Keep routes clean and semantic
- Use layout components to maintain consistency
- Use partials/components for repeated sections instead of duplicating markup

### Performance Rules

- Optimize all images
- Use responsive image sizing
- Minimize hydration
- Lazy-load below-the-fold media when appropriate
- Avoid unnecessary third-party scripts
- Keep Lighthouse performance and accessibility scores high

### Accessibility Rules

- Maintain strong contrast
- Keep body text at 16px or larger
- Provide visible focus states
- Use semantic HTML first
- Provide alt text for all meaningful images
- Respect `prefers-reduced-motion`

## SEO Rules

### Default SEO Requirements

Every page must include:

- Unique title
- Unique meta description
- Canonical URL
- Open Graph tags
- Twitter card tags if configured
- Meaningful heading hierarchy
- Descriptive alt text
- Internal links where relevant

### JSON-LD Rule

Whenever a **new page** or **new blog post** is added, always include the appropriate **JSON-LD structured data**.

This is mandatory.

### Structured Data Guidelines

Use the right schema type depending on the page:

- `Hotel`
- `LodgingBusiness`
- `WebSite`
- `WebPage`
- `BreadcrumbList`
- `Article`
- `BlogPosting`
- `FAQPage`
- `LocalBusiness`
- `Organization`

### Minimum SEO JSON-LD Practice

- Add site-wide organization or website schema where relevant
- Add page-specific schema to each major page
- Add article schema to blog posts
- Add breadcrumb schema where applicable
- Add FAQ schema when FAQ content is present
- Keep structured data aligned with visible page content

### Blog Post SEO Rules

Every new blog post should include:

- Title
- Description
- Publish date
- Modified date
- Author
- Featured image
- Canonical URL
- Article or BlogPosting JSON-LD
- Internal links to related content
- Clear headings and readable paragraphs

## Development Rules for Cursor

Cursor should follow these rules when generating or editing the project:

1. Keep the design minimal, flat, and luxury-focused
2. Use Astro best practices first
3. Use Tailwind CSS v4 for styling
4. Use Alpine.js only when lightweight interactivity is needed
5. Use GSAP only when premium motion meaningfully improves the experience
6. Keep layouts spacious and typography-led
7. Avoid unnecessary JavaScript
8. Avoid visual clutter and too many colors
9. Reuse components instead of duplicating code
10. When creating a new page or blog post, always add the necessary JSON-LD for SEO
11. Ensure all pages include solid metadata and semantic structure
12. Keep performance and accessibility as first-class requirements
